## Phase 2/3 — после прохода по табам и страницам (актуально)

Сделано: Phase 1 (Шаги 1-12, базовая миграция), Phase 2 (Шаги 13-19, табы), Phase 3 (Шаги 20-30, sidebar + полировка карточек). Что осталось обсудить с дизайнером:

### A. Зависят от бэка (требуют согласования что нужно)

Backend-тикеты живут в `docs/roadmap.md` Phase D. Нужно подтвердить что данные действительно требуются:

1. **Multi-level orderbook (5+5 уровней цен)** — для handoff `TradesTab` (стакан с горизонтальными барами). Сейчас live показывает только best bid/offer + агрегаты. **Дизайнер**: точно нужен полный стакан или достаточно агрегатов?
2. **Trades feed (лента сделок)** — для handoff `TradesTab` правая колонка. У нас вообще отсутствует. **Дизайнер**: насколько критично? Альтернатива — последняя сделка как single info-row.
3. **`yield_history` в `/bonds/{secid}/history`** — для mode-toggle Цена/Доходность в `BondHistoryTab`. Сейчас график только Цена. **Дизайнер**: оставляем только цену или ждём бэк?
4. **OFZ benchmark на графике** — пунктирная линия. **Дизайнер**: декоративно или важно?

### B. Дизайн отсутствует — нужна спецификация

Места, где у нас уже есть код но нет handoff-макета. Сейчас работает в стиле базовых токенов, но не «по дизайну». Запрос к дизайнеру:

5. **`pages/index.vue` (флэт-таблица)** — handoff даёт только grouped (`/bonds/by-issuer`). Сейчас redirect → by-issuer, флэт-таблица переехала на `/bonds/flat`. Нужен ли отдельный плоский вид? Если да — макет.
6. **`/login`, `/chat`, `/tools`, `/favorites`** — handoff не покрывает. Нужны макеты, особенно `/chat` (AI-чат с историей сессий и агентами).
7. **Skeleton-состояния** — handoff не показывает. Какие табы / экраны должны иметь skeleton при загрузке?
8. **`BondYieldsTab` (Доходности)** — handoff не имеет такого таба, у нас он существует с богатыми данными (closeyield/effectiveyield/yieldatwaprice/yieldtoprevyield + bars + comparison). **Дизайнер**: нужен ли таб совсем или мерж в Торговля? Если оставить — макет.
9. **`BondDetailsTab` (Детали)** — то же. Расширенные MOEX-поля (mid_price, marketprice, prevwaprice, dohod-данные). **Дизайнер**: нужен ли таб или это дублирование Торговли/Купонов?
10. **`BondExternalTab` (Внешние)** — iframes на сторонние ресурсы (smart-lab, dohod). Дизайн или удалить?

### C. Спорные дизайн-решения — ждём согласования

11. **Sidebar содержание из handoff** — handoff показывает 3 секции: «Каталог», «Портфель» (Мои бумаги, Алерты), «Инструменты» (Плоский список, Экспорт). У нас сейчас:
    - Каталог: Эмитенты / Месячные купоны / AI-чат
    - Инструменты: Плоский список / Утилиты
    - Скрыты: Портфель целиком (нет фичи Алертов / Экспорта)
    - Bottom: Войти / Тёмная тема
    
    **Дизайнер**: добавляем ли пункты «Алерты» / «Экспорт» (плейсхолдеры или удалить из дизайна)?

12. **`IssuerProfile` quality bars vs handoff** — handoff показывает 4 бара: Кредитное качество / Финансовая стабильность / **Прозрачность** / **Корп. управление**. У нас в dohod-данных нет полей «Прозрачность» и «Корп. управление». Маппим в:
    - Кредитное качество → `dohod.quality` (есть)
    - Финансовая стабильность → `dohod.stability` (есть)
    - Баланс → `dohod.quality_balance` (отдалённо ≠ Прозрачность)
    - Прибыльность → `dohod.quality_earnings` (отдалённо ≠ Корп. управление)
    
    **Дизайнер**: ок такие лейблы или скрыть последние два бара? Если нужны handoff-лейблы — нужен новый источник данных.

13. **Issuer header tags** — handoff макет у Sber показывает теги «Системно-значимый / Гос. участие 50%+ / Уровень 1». У нас этих данных в API нет, `props.tags` пустой массив. **Дизайнер**: источник этих сигналов или скрываем теги в Issuer header?

14. **`IssuerRating` без outlook** — handoff показывает у каждого рейтинга outlook (Стабильный / Позитивный / Негативный) с цветной меткой. У нас в `model.IssuerRating` нет поля `outlook`. **Дизайнер**: нужен outlook? Бэк-тикет: парсить и хранить.

15. **Card head на `/bonds/by-issuer` IssuerCardGrid** — handoff `icard-meta` кладёт ID-pill + ratings + AiPill в **одну** плотную строку. У нас id рядом с name, ratings отдельной строкой, AiScore там же. **Дизайнер**: ваш текущий вариант или handoff-стиль?

16. **`/issuers/[id]` таблица бумаг vs expanded на `/bonds/by-issuer`** — это два места показа бумаг эмитента. На by-issuer expanded — rich-карточки (4 metrics + 5 dates). На `/issuers/[id]` — простая таблица (SECID/Название/Погашение/Купон/Цена/YTM). **Дизайнер**: унифицировать или эти два места намеренно разные (таблица vs карточки)?

### D. Палитра / темы

17. **Палитры из handoff `tweaks-panel`** — handoff поддерживает 4 палитры (violet/blue/teal/graphite) × 2 темы = 8 комбинаций. У нас зафиксирована violet. **Дизайнер**: будем ли давать пользователю выбор палитры или одна violet навсегда?

18. **Палитра по умолчанию для разных типов бумаг?** — например ОФЗ → blue, корпоративные → violet, ВДО → orange. Просто идея, не из handoff.

### E. Минорное

19. **Иконки** — handoff DESIGN.md рекомендует Bootstrap Icons (54 уникальных, 78 вхождений). Миграция на Lucide — отдельный тикет «по желанию». Решаем после релиза.

20. **Outlier-значения в `current_yield`** — у некоторых ВДО `current_yield = 74.63%` (выбросы из-за того что цена бумаги 20% от номинала). **Дизайнер**: клампать (>200% → «—») или показывать как есть?

21. **Пороги шкалы `AiScore`** — компонент рендерит цветовые зоны по диапазонам (s-low <60 / s-med 60-79 / s-high ≥80). **Дизайнер**: проверить пороги? И/или плавный градиент вместо дискретных зон?

### F. Mobile

22. **Mobile UX в целом** — пользователь в чате отметил что «mobile далеко не на финише». Сейчас:
    - Sidebar превращается в drawer через hamburger ✓
    - KPI-сетки на bond-detail схлопываются в 2 кол ✓
    - Карточки эмитента на by-issuer — single column ✓
    - Дальше не углублялись.
    
    **Дизайнер**: нужен отдельный mobile-spec (high-fidelity макеты под 414×896)?
