# NLA · Design System

Краткий обзор визуальной системы редизайна. Подробности по миграции — `MIGRATION.md`. Готовые токены — `tokens.css`. Реализация — `components/*.vue`.

---

## 1. Принципы

1. **Данные — главный визуал.** Цифры, рейтинги, графики держат ритм страницы. Иллюстрации, иконки и эмодзи — вспомогательные, не несут смысла на скрин-уровне.
2. **Спокойная палитра.** Тёплый off-white в light, тёплый почти-чёрный в dark. Один акцент — фиолетовый. Зелёный/красный только для семантики (доходность, изменение цены, статус).
3. **Контракт `--nla-*` неизменен.** Существующие имена не переименовываем. Меняем значения, добавляем новые под тем же префиксом. Bootstrap 5 живёт.
4. **Темы равноправны.** Каждый компонент проектируется так, чтобы работать в обеих темах из одного шаблона. Контраст текста — минимум AA (4.5:1).
5. **Шрифты разделены ролями.** Inter — UI, заголовки, лейблы. JetBrains Mono — числа (tabular-nums), коды бумаг, даты, JSON-превью. Никогда не миксуем в одном слове.

---

## 2. Палитра

Все значения зашиты в `tokens.css`. Здесь — навигация по ролям.

### 2.1 Акцент · violet

| Роль | Light | Dark |
|---|---|---|
| `--nla-primary` | `#5b3aa8` | `#b89cff` |
| `--nla-primary-hover` | `#4a2e8c` | `#d0baff` |
| `--nla-primary-light` (фон под акцентом) | `#ede8f8` | `#2a1e54` |
| `--nla-primary-ink` (текст по `-light`) | `#3b2480` | `#b89cff` |

> В dark `primary-light` — это глубокий violet, **не** светлый. Это намеренно: «soft» в темноте читается как «приглушённая поверхность», а не как «осветлённый акцент».

### 2.2 Семантика

| Роль | Light | Dark | Использование |
|---|---|---|---|
| `--nla-success` | `#0d7a4e` | `#38c172` | YTM, рост цены, выплачен купон, оферта (вторично) |
| `--nla-danger`  | `#c2410c` | `#ef6c4e` | падение цены, сделка SELL, «не приобретать» |
| `--nla-warning` | `#b45309` | `#e8b75d` | спред benchmark, предупреждения |

> Зелёный — финансовый, не «весёлый». Красный — сдвинут в сторону охры, чтобы не вызывать «алерт-усталость» в плотных таблицах.

### 2.3 Поверхности

| Роль | Light | Dark | Где |
|---|---|---|---|
| `--nla-bg` | `#faf8f5` | `#0c0e0d` | фон страницы |
| `--nla-bg-card` | `#ffffff` | `#161917` | поверхность карточки |
| `--nla-bg-elevated` (новый) | `#fbf9f6` | `#1d211e` | шапка панели, второй слой внутри карточки |
| `--nla-bg-subtle` (новый) | `#f3efea` | `#131614` | nav, неактивные сегменты, фон бейджей |
| `--nla-bg-hover` | `#f3efea` | `#1d211e` | hover-стейты |

### 2.4 Текст · AA-контраст

| Роль | Light на `--nla-bg` | Dark на `--nla-bg` | Использование |
|---|---|---|---|
| `--nla-text`           | 14.7:1 | 14.6:1 | основной |
| `--nla-text-secondary` | 9.1:1  | 8.9:1  | подзаголовки, описания |
| `--nla-text-muted`     | 4.6:1  | 5.4:1  | мета, лейблы полей |
| `--nla-text-subtle` (новый) | 3.2:1 | 3.6:1 | **только** для текста ≥18px (даты-плейсхолдеры, фоновые меты) |

### 2.5 Цвета рейтингов — **не меняем**

Tier-палитра в `composables/useRating.ts::ratingTierStyle` (AAA→зелёный, AA→синий, A→teal, BBB→жёлтый, BB→оранжевый, B/CCC/CC/C/D→красный) и AI-палитра в `useFormat.ts::aiRatingStyle` (5 порогов 0-100) — часть продуктовой логики. Дизайн строится **вокруг** них, не заменяет их.

## 3. Типографика

```
--nla-font:      'Inter',          system-ui, sans-serif    /* UI */
--nla-font-mono: 'JetBrains Mono', ui-monospace, monospace  /* числа, коды, JSON */
```

Шкала (используется напрямую, без CSS-переменных — слишком быстро меняется):

| Token | px / line | Где |
|---|---|---|
| display | 24 / 32, 700, -0.02em | название бумаги в hero |
| h2 | 18 / 24, 700, -0.01em | заголовок секции, имя эмитента |
| h3 | 13 / 18, 600 | заголовок панели |
| body | 14 / 21, 400 | основной |
| body-strong | 13 / 18, 500 | значения в info-row |
| caption | 12 / 16, 400 | вторичный текст |
| micro | 10.5 / 14, 600, 0.06em letter-spacing, uppercase | лейблы KPI, меты, табличные шапки |
| mono-num | 14 / 18, 500 mono, tabular-nums | цены, объёмы, доходности |
| mono-num-lg | 24 / 28, 600 mono, -0.02em | KPI-значения |

**Числа.** Везде, где встречается число — `font-feature-settings: 'tnum', 'zero'` (или класс `.num`). Запятая как разделитель (`13,42%`, `1 000,00 ₽`), пробел как тысячный разделитель (`50 000 000 шт.`).

## 4. Spacing & Radius

Spacing на основе шага 4px. Используем токены `--nla-space-1..10` или Bootstrap `g-*` / `p-*` (они выровнены).

| Token | px | Где |
|---|---|---|
| `--nla-space-1` | 4  | gap внутри пилл, между точкой и текстом |
| `--nla-space-2` | 8  | стандартный gap между inline-элементами |
| `--nla-space-3` | 12 | gap между карточками внутри секции |
| `--nla-space-4` | 16 | стандартный padding панели |
| `--nla-space-5` | 20 | hero-padding по вертикали |
| `--nla-space-6` | 24 | главный отступ страницы |
| `--nla-space-8` | 32 | между секциями |
| `--nla-space-10`| 40 | hero-padding по горизонтали на десктопе |

| Radius | px | Где |
|---|---|---|
| `--nla-radius-sm`  | 6  | бейджи, теги, маленькие кнопки |
| `--nla-radius`     | 10 | input, badge крупный, sidebar item |
| `--nla-radius-lg`  | 14 | панель |
| `--nla-radius-xl`  | 20 | модалка |
| `--nla-radius-2xl` (новый) | 28 | hero, focus-карточка |
| `--nla-radius-pill` (новый) | 999 | прогресс-бар, статус-точка, pill-кнопка |

| Token | Где |

| `--nla-shadow`    | hover карточки эмитента, активный таб |

## 8. Что **не** трогаем

- Bootstrap 5: `card`, `card-body`, `row g-*`, `col-*`, `btn`, `form-control`, `input-group`, `dropdown`, `modal`, `nav-tabs`, BS-таблицы. Перебиваем токенами и кастомными классами через `!important` (так уже сделано) — не выпиливаем.
- API-контракт. Backend стабильный, дизайн чисто фронтовый.
- `composables/useRating.ts` и `useFormat.ts` — палитры рейтингов и AI-скоров. Часть продуктовой логики.
- Конфиги Chart.js по компонентам. Меняем значения `--nla-chart-1..6`, не трогаем код графиков.
- Существующие кастомные классы (`panel-header`, `info-row`, `filter-panel`, `yield-bar`, `issuer-bond-metrics`) — не переименовываем, могут быть захардкожены в шаблонах.
- «Мёртвые» секции в `main.css` (переопределения `bg-light`, `border-bottom`, `.text-success !important`) — оставляем, пока не сломают новый дизайн.

---

## 9. Доступность

- Контраст текст/фон: AA (4.5:1) минимум для body, 3:1 для текста ≥18px.
- Все интерактивные элементы имеют `:focus-visible` обводку: `outline: 2px solid var(--nla-primary); outline-offset: 2px`.
- Цвет — никогда не единственный носитель смысла. Статус «выплачен/ожидается» дублируется иконкой + текстом, а не только зелёным/серым.
- В тёмной теме `--nla-text-muted` = `#919a91` (5.4:1) — это нижняя граница для body. Не опускать.
